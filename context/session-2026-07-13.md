## 2. Phase 0 ‚Äî Setup ‚úÖ

Scaffold, dependencies, and gitignore were already in place. The remaining work was the
API key, which produced the session's most instructive bug.

### ‚öÝÔ∏è The stale-key gotcha (worth understanding)

Symptom: the app returned `{"error":"Server is missing a valid API key."}` (HTTP 500) even
though the key in `.env.local` was valid and worked when tested directly with `curl`.

Cause: **`~/.zshrc` exported a stale, deleted `ANTHROPIC_API_KEY` globally, and Next.js lets
a real shell environment variable override `.env.local`.** The app was sending the dead key.

Fix applied:
1. Removed the global `export ANTHROPIC_API_KEY=...` line from `~/.zshrc`.
2. Keys now live **only** per-project in `.env.local`.

Note: an already-open terminal keeps the old value in memory. In an existing shell run
`unset ANTHROPIC_API_KEY` (or open a new terminal) before `npm run dev`.

**Concept to be able to explain:** the key is read server-side only. Every LLM call goes
through a Next.js route handler, so the key never reaches the browser.

## 3. Phase 1 ‚Äî First LLM call ‚úÖ

### Files

| File | Purpose |
|---|---|
| `lib/anthropic.ts` | Shared Anthropic client + `MODEL`, `MAX_INPUT_CHARS` (50,000), `REQUEST_TIMEOUT_MS` (60s) |
| `app/api/summarize/route.ts` | `POST` ‚Äî validate ‚Üí call Claude ‚Üí return summary + token usage |
| `app/page.tsx` | Textarea ‚Üí Summarize button ‚Üí summary, with loading, char counter, error states |

### Key technical decisions

- **No `temperature`.** Older guidance recommends a low temperature (~0.2) for factual
  summarization. That is now **out of date** ‚Äî `temperature`, `top_p`, and `top_k` are
  *removed* on current models (Opus 4.8, Sonnet 5) and return a **400 error**. Factual,
  no-speculation tone is steered entirely through the system prompt instead.
- **Model:** `claude-opus-4-8`, set in one constant in `lib/anthropic.ts`. Cost is
  $5/$25 per 1M tokens in/out ‚Äî swap to `claude-haiku-4-5` ($1/$5) for cheap iteration.
- **Input cap:** 50,000 chars (~12‚Äì15k tokens). Oversized input is rejected with a 413 that
  points at retrieval as the real answer ‚Äî that is Phase 4.
- **Typed error handling**, most-specific first: `AuthenticationError` ‚Üí 500,
  `RateLimitError` ‚Üí 429, `APIConnectionError` (incl. timeouts) ‚Üí 504, `APIError` ‚Üí 502.
- **Token usage returned** in the response ‚Äî groundwork for the Phase 5 cost guardrails.
